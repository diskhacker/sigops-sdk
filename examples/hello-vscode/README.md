# hello-vscode

A pointer example for the [@sigops/vscode-ext](../../packages/vscode-ext)
VS Code extension. Unlike the other `hello-*` examples, this one is
**text-only**: the extension runs inside VS Code, not inside a Node process,
so there's nothing to `execute`.

## What this example shows

1. Where the extension source lives in this repo (`packages/vscode-ext`).
2. How to build a `.vsix` from source.
3. How to install the extension into VS Code (both from source and from the
   Marketplace once it's published).
4. What behaviour to expect after installing.

The compiled module exports a single constant, `VSCODE_INSTALL_INSTRUCTIONS`,
so downstream tooling (docs builder, onboarding scripts, `sigops doctor`)
can print the canonical text without copy-pasting.

## Install & build

From the repo root:

```bash
pnpm install
pnpm --filter @sigops-examples/hello-vscode build
```

## Use it

```ts
import { getVSCodeInstallInstructions } from '@sigops-examples/hello-vscode';

console.log(getVSCodeInstallInstructions());
```

## Installing the VS Code extension

### Option A — from source (this repo)

```bash
pnpm install
pnpm --filter @sigops/vscode-ext build
cd packages/vscode-ext
npx vsce package            # produces sigops-vscode-ext-*.vsix
```

Then in VS Code: Extensions panel -> **...** menu -> *Install from VSIX...*
and pick the file.

### Option B — from the VS Code Marketplace

Once published:

1. Open VS Code.
2. Open the Extensions panel (`Ctrl+Shift+X` / `Cmd+Shift+X`).
3. Search for **SigOps SEL**.
4. Click **Install**.

## What you get

- Syntax highlighting for `*.sel` files (TextMate grammar at
  `packages/vscode-ext/syntaxes/sel.tmLanguage.json`).
- LSP-backed completions, hover, and diagnostics — powered by
  `@sigops/sel-tools`'s `SELLanguageServer`.
- Format-on-save using the canonical `format()` from `sel-tools`.

## Smoke test

1. Install the extension.
2. Open any `.sel` file (e.g. one generated by `hello-template`).
3. Confirm `monitor`, `alert`, `action`, and `when` are coloured as keywords.
4. Introduce a syntax error (unclosed `{`); confirm you see a red squiggle.
5. Right-click -> *Format Document*; confirm the file is re-indented.
